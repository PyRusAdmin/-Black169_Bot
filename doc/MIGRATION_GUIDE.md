# Инструкция по миграции данных и использованию системы уровней

## 🚀 Быстрый старт

### 1. Миграция данных из JSON в БД

Для переноса данных о клиентах из `data/clients_levels.json` в базу данных выполните:

```bash
python migrate_clients_to_db.py
```

**Что делает скрипт:**

- Загружает данные из JSON файла (7389 клиентов)
- Нормализует телефонные номера (приводит к формату `79999999999`)
- Ищет пользователей в БД по телефону или ID QuickResto
- Обновляет поля `client_level` и `accumulation_amount`

**Пример результата:**

```
============================================================
📊 ОТЧЕТ О МИГРАЦИИ ДАННЫХ
============================================================
📁 Всего клиентов в JSON: 7389
✅ Обновлено в БД: 4
❌ Не найдено в БД: 7385
⚠️  Ошибок: 0
============================================================
📈 Процент успешных: 0.1%
============================================================

✅ Миграция успешно завершена!
```

> **Примечание:** В БД обновляются только те клиенты, которые зарегистрированы в боте (отправили номер телефона).

## 🔄 Синхронизация с QuickResto

Для обновления данных из QuickResto и синхронизации с БД:

1. В админ-панели нажмите **"🔄 Анализ и синхронизация клиентов"**
2. Дождитесь завершения процесса
3. Получите отчет с результатами

**Что происходит:**

1. Загрузка всех клиентов из QuickResto API
2. Нормализация телефонных номеров
3. Определение уровней по накопительной сумме
4. Сохранение в `data/clients_levels.json`
5. Синхронизация с базой данных

## ❓ Частые вопросы

### Почему в БД обновилось только 4 клиента из 7389?

В базе данных хранятся только те пользователи, которые зарегистрировались в боте (отправили номер телефона через
`/start`).

### Как обновить данные для всех клиентов?

Используйте кнопку **"🔄 Анализ и синхронизация клиентов"** в админ-панели.

### Можно ли добавить уровни для пользователей, которых нет в боте?

Нет, уровни определяются только для зарегистрированных пользователей. Для остальных данные хранятся в JSON файле и
отображаются в общей статистике.

### Как часто нужно запускать синхронизацию?

Рекомендуется запускать синхронизацию:

- После крупных мероприятий/акций
- Раз в неделю для актуализации данных
- При необходимости получить свежую статистику
