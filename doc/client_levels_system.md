# Механизм определения уровня клиента

## Описание системы уровней

Система уровней клиентов основана на **накопительной сумме** (accumulationBalance) из QuickResto API.

## Уровни клиентов

| Уровень       | Мин. сумма  | Описание                  |
|---------------|-------------|---------------------------|
| 💎 **Black**  | от 60 000 ₽ | Привилегированный уровень |
| 🥇 **Gold**   | от 30 000 ₽ | Золотой уровень           |
| 🥈 **Silver** | от 10 000 ₽ | Серебряный уровень        |
| 🥉 **Bronze** | от 0 ₽      | Базовый уровень           |

## Алгоритм определения уровня

```python
def get_level(accumulation: float) -> str:
    """
    Определение уровня клиента по накопительной сумме.
    
    :param accumulation: Накопительная сумма клиента
    :return: Название уровня (Black, Gold, Silver, Bronze)
    """
    for level in LEVELS:
        if accumulation >= level["min_amount"]:
            return level["name"]
    return "Bronze"
```

Алгоритм работает следующим образом:

1. Получает накопительную сумму клиента из QuickResto API
2. Последовательно проверяет пороги уровней от высшего к низшему
3. Возвращает первый уровень, порог которого превышен
4. Если ни один порог не превышен — возвращает "Bronze"

## Источник данных

Данные о накопительной сумме получаются из QuickResto API через эндпоинт:

```
GET {layer_name}.quickresto.ru/platform/online/api/read
```

Поле в ответе API:

```json
{
  "accumulationBalance": {
    "ledger": 150000.00
    // ← это значение используется для определения уровня
  }
}
```

## Синхронизация данных

### 1. Автоматическая синхронизация через админ-панель

Администратор может запустить полную синхронизацию через кнопку:
**"🔄 Анализ и синхронизация клиентов"**

Процесс включает:

1. Загрузка всех клиентов из QuickResto API
2. Нормализация телефонных номеров (приведение к формату `79999999999`)
3. Определение уровня для каждого клиента
4. Сохранение в `data/clients_levels.json`
5. Синхронизация с базой данных (обновление полей `client_level` и `accumulation_amount`)

### 2. Программный вызов

```python
from services.quickresto_api import analyze_and_sync_clients

# Запуск полного цикла синхронизации
result = analyze_and_sync_clients()

# Результат содержит:
# - total_clients: всего клиентов
# - json_saved: успешно ли сохранён JSON
# - db_sync_stats: статистика синхронизации с БД
# - level_distribution: распределение по уровням
# - timestamp: время выполнения
```

## Нормализация телефонных номеров

Для приведения телефонов к единому виду используется функция `normalize_phone_number()`:

**Входные форматы:**

- `+7 924 111-14-44`
- `89241111444`
- `+79241111444`
- `8 924 111 14 44`

**Выходной формат:**

- `79241111444` (11 цифр, начиная с 7)

## Хранение данных в БД

В таблицу `registered_persons` добавлены поля:

| Поле                  | Тип  | Описание                                   |
|-----------------------|------|--------------------------------------------|
| `client_level`        | TEXT | Уровень клиента (Black/Gold/Silver/Bronze) |
| `accumulation_amount` | REAL | Накопительная сумма для определения уровня |

## Обновление данных

### При регистрации пользователя

Когда пользователь регистрируется в боте, его уровень автоматически определяется и сохраняется:

```python
# В handlers/user_handlers.py при регистрации
data = {
    "id_telegram": id_telegram,
    "client_level": full_data.get("level"),  # Уровень из QuickResto
    "accumulation_amount": full_data.get("accumulation_balance", {}).get("ledger", 0),
    # ... другие поля
}
write_to_db_registered_person(data)
```

### При синхронизации

Массовое обновление уровней происходит при запуске `analyze_and_sync_clients()`:

- Обновляются все зарегистрированные пользователи
- Поиск по телефону или ID QuickResto
- Обновляются поля `client_level` и `accumulation_amount`

## Механизм работы (пошагово)

1. **Администратор нажимает "Анализ и синхронизация клиентов"**
   ↓
2. **Загрузка данных из QuickResto API**
    - GET /api/list — получение списка ID клиентов
    - GET /api/read — получение полных данных для каждого клиента
      ↓
3. **Обработка данных**
    - Извлечение накопительной суммы (accumulationBalance.ledger)
    - Определение уровня через `get_level()`
    - Нормализация телефона через `normalize_phone_number()`
      ↓
4. **Сохранение в JSON**
    - Сортировка по накопительной сумме (по убыванию)
    - Запись в `data/clients_levels.json`
      ↓
5. **Синхронизация с БД**
    - Поиск пользователя по телефону в `registered_persons`
    - Обновление полей `client_level` и `accumulation_amount`
      ↓
6. **Формирование отчета**
    - Статистика по уровням
    - Количество обновлённых записей
    - Ошибки и не найденные пользователи
